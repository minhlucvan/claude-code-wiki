## この Wiki について

**Claude Code Wiki** は、Claude Code のアーキテクチャ、エンジニアリングパターン、競争優位性を理解するための決定版ガイドです。**512,000行の本番 TypeScript コード**の分析を通じて、この wiki は以下を明らかにします：

- **10のアーキテクチャイノベーション**により、Claude Code は競合他社を上回る
- **ストリーミングツール実行**により、LLM がストリーミング中にツールを実行（2〜5倍高速な UX）
- **5層コンテキスト管理**により、無制限の会話メモリを実現
- **マルチエージェントオーケストレーション**とキャッシュ共有（90%のコスト削減）
- **React ターミナル UI** により、CLI で本番グレードの UX を提供
- **AST レベルのセキュリティ**による深いコマンド分析（正規表現ではない）
- **本番エンジニアリングパターン**は、フリート規模の経済性のために最適化

**これは単なる AI コーディングツールではありません** — Claude を作成したチームによって構築され、競合他社が持たないファーストパーティ API アクセスと最適化の機会を持っています。

## この Wiki が存在する理由

この wiki は、Claude Code を優れたものにする本番グレードのパターンとアーキテクチャの決定を文書化するために存在します。競合他社が苦労している難しい問題をどのように解決するかを学びます：

- **速度**：ほとんどのツールは、ツールを順次実行する前に LLM の完了を待ちます。Claude Code は、ストリーミング中にツールを同時実行し、2〜5倍高速なマルチツール操作を実現します。
- **メモリ**：競合他社は基本的なコンテキストの切り捨てを使用するか、手動クリーンアップを必要とします。Claude Code は、無制限の会話のために5層の自動圧縮パイプラインを使用します。
- **コスト**：複数のエージェントの実行はコストがかかります。Claude Code のキャッシュフォーク最適化は、共有キャッシュを通じて90%のコスト削減を達成します。
- **セキュリティ**：ほとんどのツールは、コマンド解析に正規表現を使用します。Claude Code は、深いセキュリティ分析のために AST レベルの Bash 解析を使用します。
- **スケール**：フリート規模の経済性のために構築され、組織レベルで Gtok/週を最適化します。

この wiki は、これらのパターンとテクニックを文書化しているので、独自の AI ツールに学び、適用できます。
